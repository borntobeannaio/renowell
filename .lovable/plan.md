

## Исправление навигации по уведомлениям к задачам

### Корневая причина

При создании уведомлений о задачах (назначение, упоминание) поле `link` не заполняется -- оно всегда `null`. Это вызывает две проблемы:

1. **Telegram / Email**: Edge-функция `send-external-notification` строит ссылку из `notification.link`. Раз оно пустое, в Telegram и письмах нет кликабельной ссылки вообще.
2. **Внутренние уведомления**: `NotificationItem` использует запасной вариант через `related_task_id` и переходит на `/tasks?task=ID`. Но `TasksModule` ожидает, что задачи уже загружены и не отфильтрованы, из-за чего возникает гонка состояний.

### Решение

Добавить явное поле `link` при создании всех уведомлений о задачах.

### Файлы для изменения

**1. `src/hooks/useTasks.ts`** -- уведомления о назначении задачи

Добавить `link: "/tasks?task=" + newTask.id` в каждый объект уведомления (строка ~98-103):

```
link: `/tasks?task=${newTask.id}`,
```

**2. `src/hooks/useTaskComments.ts`** -- уведомления об упоминании

Добавить `link: "/tasks?task=" + taskId` в каждый объект уведомления (строка ~91-96):

```
link: `/tasks?task=${taskId}`,
```

### Результат

- Telegram-сообщения и email будут содержать рабочую ссылку вида `https://renowell.silkagro.ru/tasks?task=UUID`
- Клик по уведомлению в приложении будет использовать `link` напрямую (а не запасной `related_task_id`), что надёжнее
- `TasksModule` уже обрабатывает параметр `?task=ID` корректно -- сбрасывает фильтры и открывает модалку

### Технические детали

Изменения минимальные -- по одной строке в двух файлах. Логика `NotificationItem` и `send-external-notification` уже умеет работать с полем `link`, просто сейчас оно не заполняется.
