

## Упрощение интерфейса календаря + подключение к данным

### Проблема

Текущий календарь перегружен: 4 типа фильтров, переключатель месяц/неделя, недельная сетка с временной шкалой, блок "Ближайшие события" внизу, цветовые категории -- и все это на моковых данных. Непонятно, что делать и зачем.

### Новый подход: простой и понятный

Убираем лишнее, оставляем суть:

```text
+----------------------------------------------+
|  Календарь              [+ Новая встреча]     |
+----------------------------------------------+
|                                               |
|  [< Февраль 2026 >]                          |
|                                               |
|  Пн Вт Ср Чт Пт Сб Вс                       |
|  ...  ...  ...  ...  ...  ...  ...            |
|  ... [15] ...  ...  ...  ...  ...             |
|  ...  ...  ...  ...  ...  ...  ...            |
|                                               |
+----------------------------------------------+
|  Сегодня, 15 февраля                          |
|                                               |
|  10:00 - 11:00  Статус проекта     Перег. А   |
|  14:00 - 14:30  1-on-1             Онлайн     |
|  16:00 - 17:00  Тех. обзор         Онлайн     |
|                                               |
|  Нет событий? -- "Пока пусто"                 |
+----------------------------------------------+
```

### Что убираем

- Фильтры по типам (Командные / Личные / Проектные / Внешние) -- пока нет реальных данных, они бессмысленны
- Недельный вид с временной шкалой (сложный, почти неюзабельный на мобильных)
- Блок "Ближайшие события" внизу (дублирует список дня)
- Переключатель Месяц/Неделя

### Что оставляем / добавляем

1. **Календарь-сетка** -- компактный, с точками на днях с событиями
2. **Список событий выбранного дня** -- простые карточки: время, название, место/онлайн, участники
3. **Кнопка "Новая встреча"** -- открывает модалку создания (из утвержденного плана)
4. **Блок "Ближайшие"** -- компактный список 3-5 ближайших встреч под календарем (только если выбранный день пуст)

### Что включено из утвержденного плана

Одновременно с упрощением UI реализуем полный функционал:

1. **Таблица `calendar_events`** в БД (title, start_time, end_time, location, is_online, creator_id, participant_ids)
2. **Edge-функция `send-calendar-invite`** -- генерация ICS и отправка email через Resend
3. **Хук `useCalendarEvents`** -- CRUD через dbProxy
4. **Модалка создания встречи** -- название, дата/время, участники (выбор из сотрудников), место
5. **Возврат в навигацию** -- пункт "Календарь" в Sidebar и MobileNav
