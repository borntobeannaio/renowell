

# Исправление бага: ложное срабатывание доступа к протоколам

## Проблема
В `useProtocolPermissions.ts` строка 70: когда у протокола `organizer` равен `null`, он становится пустой строкой `""`. Метод `String.includes("")` всегда возвращает `true`, поэтому любой сотрудник с именем получает доступ.

Аналогичная проблема может быть и с пустыми attendees (строка 65-69): если attendee — пустая строка, `variant.includes("")` тоже `true`.

## Решение

**Файл:** `src/hooks/useProtocolPermissions.ts`

1. Добавить проверку на пустую строку перед `includes`:
   - Строка 63: пропускать пустые attendees: `if (!a) return false` 
   - Строка 70: пропускать пустой organizer: добавить `organizer.length > 0` в условие

Минимальное изменение — 2 строки.
