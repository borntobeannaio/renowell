
# План: Добавить редактирование сотрудников для HR-администраторов

## Что будет сделано

HR-администраторы (`sonya369@gmail.com` и `astashkina495@gmail.com`) смогут редактировать данные сотрудников прямо из карточки. При сохранении изменений будет обновляться связанный профиль пользователя, а триггер в базе данных автоматически синхронизирует данные обратно в таблицу сотрудников.

## Изменения в интерфейсе

1. В карточке сотрудника для HR-администраторов появится кнопка "Редактировать"
2. При нажатии откроется форма редактирования с полями:
   - ФИО
   - Должность
   - Телефон
   - Дата рождения
3. После сохранения карточка обновится с новыми данными


## Технические детали

### 1. Новый компонент `EditEmployeeModal.tsx`

Создаём модальное окно для редактирования сотрудника:
- Поля: ФИО, Должность, Телефон, Дата рождения
- Загрузка данных профиля по `profile_id` сотрудника
- При сохранении:
  1. Обновляем `profiles` (триггер синхронизирует в `employees`)
  2. Отдельно обновляем `phone` в `employees` (этого поля нет в профиле)

### 2. Изменения в `HRModule.tsx`

- Добавляем кнопку "Редактировать" в карточку сотрудника (видна только HR-админам)
- Подключаем `EditEmployeeModal`
- После успешного редактирования обновляем список сотрудников

### 3. Расширение типа `DbEmployee`

Добавляем `profile_id` в интерфейс для связи с профилем при редактировании.
