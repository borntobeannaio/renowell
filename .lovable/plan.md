

## Пересылка входящих сообщений бота в Telegram @anna_sirum

### Проблема
Нужно пересылать все входящие сообщения бота администратору @anna_sirum, но её chat_id неизвестен (не сохранён в базе).

### Решение

#### Шаг 1: Добавить секрет FORWARD_CHAT_ID
- Создать новый секрет `FORWARD_CHAT_ID` (пока пустой или с заглушкой)
- Как только Анна напишет боту, её chat_id станет известен и будет автоматически сохранён

#### Шаг 2: Изменить `supabase/functions/telegram-webhook/index.ts`

Добавить логику пересылки:

1. **Автообнаружение Анны**: при получении сообщения от пользователя с username `anna_sirum` — сохранить chat_id в переменную окружения (через базу данных, т.к. секреты нельзя менять из кода). Вместо этого, использовать **таблицу настроек** или **хардкод** после первого обнаружения.

2. **Пересылка сообщений**: после обработки каждого входящего сообщения (кроме канальных постов), переслать его Анне через Telegram API `forwardMessage`.

**Упрощённый подход (без отдельной таблицы):**
- Добавить в начало обработки сообщений проверку username отправителя
- Если username === `anna_sirum`, запомнить chat_id в таблице `profiles` или отдельной таблице `bot_settings`
- Для пересылки использовать `forwardMessage` API, чтобы Анна видела оригинальное сообщение

**Итоговая логика в webhook:**
```
1. Получено сообщение
2. Если username === "anna_sirum" -> сохранить chat_id в таблицу bot_settings
3. Загрузить forward_chat_id из bot_settings
4. Если forward_chat_id существует -> вызвать forwardMessage
5. Продолжить обычную обработку (привязка, /start и т.д.)
```

#### Шаг 3: Создать таблицу `bot_settings`
```sql
CREATE TABLE bot_settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
-- RLS: только чтение для edge functions через service_role
```
Это позволит хранить `forward_chat_id` без необходимости менять секреты.

### Порядок действий пользователя
1. После деплоя — попросить Анну (@anna_sirum) написать `/start` боту @renowell_bot
2. Бот автоматически определит её chat_id и начнёт пересылать все входящие сообщения

### Технические детали
- Файл: `supabase/functions/telegram-webhook/index.ts`
- Новая таблица: `bot_settings` (key-value)
- Telegram API: `forwardMessage` для пересылки оригинальных сообщений
- Пересылаются только личные сообщения боту (не канальные посты)
- Анна продолжит получать обычные ответы бота в дополнение к пересланным сообщениям от других пользователей
