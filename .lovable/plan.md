
# План: Синхронизация задач протоколов без дублирования

## Текущая ситуация

| Метрика | Значение |
|---------|----------|
| Всего пунктов протоколов | 213 |
| Уникальных текстов пунктов | 69 |
| Дублей (скопированные пункты) | 144 |
| Пунктов с task_id | 7 |
| Пунктов без task_id | 206 |
| Уникальных пунктов, которым нужны задачи | 68 |
| "Сиротских" задач к удалению | 47 |

Пункты протоколов копируются при создании нового протокола. Один и тот же пункт "Разработать портал..." существует в 5 протоколах, но задача привязана только к последней копии.

## Правильная логика

Каждый **уникальный текст пункта** = **одна задача**. Все копии пункта в разных протоколах должны ссылаться на одну задачу.

```text
Пункт "Разработать портал"
├── Протокол #48 (28.01) → task_id = abc123 ✓
├── Протокол #47 (27.01) → task_id = abc123 (обновить)
├── Протокол #46 (21.01) → task_id = abc123 (обновить)
├── Протокол #45 (19.01) → task_id = abc123 (обновить)
└── Протокол #44 (01.12) → task_id = abc123 (обновить)
```

## Шаги миграции

### Шаг 1: Связать дубли с существующими задачами

7 уникальных текстов уже имеют задачи. Найти все копии этих пунктов и проставить им тот же task_id.

```sql
-- Обновить все дубли, чтобы они ссылались на ту же задачу
WITH existing_tasks AS (
  SELECT DISTINCT ON (item_text)
    item_text,
    task_id
  FROM protocol_items
  WHERE task_id IS NOT NULL
)
UPDATE protocol_items pi
SET task_id = et.task_id
FROM existing_tasks et
WHERE pi.item_text = et.item_text
  AND pi.task_id IS NULL;
```

### Шаг 2: Создать задачи для оставшихся 62 уникальных пунктов

Для каждого уникального текста без задачи:
1. Взять данные из **последней версии** пункта (последний протокол по дате)
2. Создать одну задачу
3. Проставить task_id всем копиям этого пункта

### Шаг 3: Удалить 47 "сиротских" задач

Задачи с `origin_type = 'protocol'`, которые не связаны ни с одним пунктом.

### Шаг 4: Обновить данные задач из актуальных версий пунктов

Синхронизировать title, assignee_ids, due_date из последней версии каждого пункта в связанную задачу.

## Техническая реализация

### Edge-функция `sync-protocol-tasks`

Создать одноразовую функцию, которая:

1. **Шаг 1**: Получить все пункты с task_id и распространить связь на дубли
2. **Шаг 2**: Найти уникальные пункты без задач, создать задачи, связать все копии
3. **Шаг 3**: Удалить сиротские задачи
4. **Шаг 4**: Обновить данные задач из последних версий пунктов

```typescript
// Псевдокод
async function syncProtocolTasks() {
  // Шаг 1: Распространить существующие связи
  await spreadExistingTaskLinks();
  
  // Шаг 2: Создать недостающие задачи
  const uniqueItemsWithoutTasks = await getUniqueItemsWithoutTasks();
  for (const item of uniqueItemsWithoutTasks) {
    const task = await createTask(item);
    await linkAllCopiesToTask(item.item_text, task.id);
  }
  
  // Шаг 3: Удалить сиротские задачи
  await deleteOrphanTasks();
  
  // Шаг 4: Синхронизировать данные задач
  await syncTaskDataFromLatestItems();
}
```

### Алгоритм определения "актуальной версии"

Актуальная версия пункта — из протокола с максимальной датой (и номером при равной дате):

```sql
SELECT DISTINCT ON (item_text)
  id, item_text, responsible, due_date, section_id
FROM protocol_items pi
JOIN protocols p ON p.id = pi.protocol_id
WHERE archived = false
ORDER BY item_text, p.date DESC, p.number DESC
```
