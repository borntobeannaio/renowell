

## Пересоздание задач из протокола от 09/02

### Что будет сделано

1. **Удаление всех 97 существующих задач** (и связанных комментариев к задачам)
2. **Создание новых задач** из всех ~35 активных пунктов протокола №48
3. **Привязка задач к протоколу** (обновление `task_id` в `protocol_items`)
4. **Назначение исполнителей и сроков** из данных пунктов протокола
5. **Пометка готовых задач** (2 пункта с `completed = true` получат статус "done")
6. **Копирование комментариев** из протокола в задачи (1 комментарий к пункту про страховку)

### Данные из протокола

- **35 активных пунктов** в 15 секциях (проекты, тендеры, HR, бизнес-задачи, цели)
- **2 выполненных пункта**: "Подписать ДС 3 в ЭДО" и "Написать письмо по страховке"
- **1 комментарий**: к пункту "Написать письмо по страховке" от Софьи Нечаевой
- Привязка к проектам через секции (УФА МЕГА, ХУАМИН, ВК входная группа и т.д.)
- Специальные проекты для секций: Тендеры, Бизнес процессы, Подбор персонала, Цели компании

### Технические детали

Будет обновлена edge-функция `sync-protocol-tasks` с новым режимом "rebuild":

1. **Удаление**: `DELETE FROM task_comments`, `DELETE FROM comment_mentions` (по task_id), затем `DELETE FROM tasks`
2. **Создание задач**: для каждого `protocol_item` с `archived = false` в протоколе №48:
   - `title` = `item_text` (для тендеров с префиксом `[company]`)
   - `assignee_ids` = profile_id сотрудников из поля `responsible` (или `default_responsible` секции)
   - `project_id` = `entity_id` секции (для проектов) или системный ID (для тендеров/бизнес/HR/целей)
   - `due_date` = из пункта протокола
   - `status` = "done" если `completed = true`, иначе "new"
   - `origin_type` = "protocol"
3. **Привязка**: обновление `task_id` в `protocol_items`
4. **Комментарии**: копирование из `protocol_item_comments` в `task_comments` с тем же `author_id` и `content`
