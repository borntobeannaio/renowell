

# Почему entity_name = "Лукойл" у тендерных секций

## Причина

Это историческая проблема. В раннем дизайне тендерных секций `entity_name` использовался для хранения названия компании (одна компания = одна секция). После редизайна тендеры стали использовать внутренние группы компаний (`[CompanyName]` префикс в item_text), а `entity_name` перестал быть значимым.

Первая тендерная секция была создана с `entity_name = "Лукойл"`. При **копировании протоколов** (строка 416 в ProtocolEditor.tsx) `entity_name` переносится из источника как есть — поэтому "Лукойл" размножился на все последующие копии.

На UI это не влияет — компонент `TenderSection` всегда показывает заголовок "Тендеры" хардкодом, а не из `entity_name`.

## План исправления

### 1. Исправить данные в базе
SQL-миграция: обновить все тендерные секции, поставив `entity_name = 'Тендеры'`:
```sql
UPDATE protocol_sections 
SET entity_name = 'Тендеры' 
WHERE section_type = 'tender' AND entity_name != 'Тендеры';
```

### 2. Защитить от повторения
В `ProtocolEditor.tsx` при сохранении тендерной секции (строка 835) принудительно использовать `"Тендеры"` вместо `group.entityName`:
```typescript
entity_name: group.sectionType === 'tender' ? "Тендеры" : (group.entityName || null),
```

Аналогичная правка при копировании (строка 416) — не нужна, т.к. после исправления данных в БД все источники уже будут иметь корректное значение, а новые секции будут создаваться с "Тендеры".
