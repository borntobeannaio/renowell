

# История изменений задач (с комментариями)

## Изменения

### 1. Database: Добавить audit-триггер на таблицу `tasks`
SQL-миграция — привязать существующую функцию `audit_trigger_func()` к таблице `tasks` на INSERT/UPDATE/DELETE. Это автоматически запишет все изменения задач в `audit_log`.

### 2. Новый компонент `src/components/tasks/TaskHistory.tsx`
Вертикальный таймлайн, объединяющий два источника данных:
- **audit_log** (где `table_name = 'tasks'`) — создание, изменения статуса/приоритета/исполнителей/срока/названия, удаление
- **task_comments** — комментарии к задачам

Логика:
- Загрузить оба набора данных параллельно (через `proxySelect`)
- Объединить в единый массив по времени (`changed_at` / `created_at`), отсортировать DESC
- Для каждой записи резолвить автора через `profiles` (кэш сотрудников из `useEmployees`)
- Diff `old_data` vs `new_data` с человекочитаемыми лейблами полей:
  - `status` → "Статус" (с маппингом значений)
  - `priority` → "Приоритет"
  - `assignee_ids` → "Исполнители" (резолв имён)
  - `due_date` → "Срок"
  - `title` → "Название"
- Комментарии отображаются как отдельный тип записи в таймлайне с иконкой сообщения и текстом комментария

Визуал: вертикальная линия слева, цветные точки (зелёный=создание, синий=изменение, красный=удаление, серый=комментарий), карточки с аватаром автора, временем и описанием изменения.
