

## Исправление дропдауна выбора участников в модалке создания встречи

### Проблема

Дропдаун `EmployeeMultiSelect` рендерится через React Portal в `document.body`. Radix Dialog перехватывает все pointer-события за пределами своего `DialogContent` и интерпретирует их как клик снаружи, что приводит к:

1. **Закрытие при вводе в поиск** -- клик по полю ввода внутри портала воспринимается Radix как `interactOutside`, и диалог/дропдаун закрывается.
2. **Невозможность скролла** -- pointer-события при скролле тоже перехватываются Radix.

### Решение

Добавить остановку распространения событий (`stopPropagation`) на контейнере дропдауна, отрендеренного через портал. Это не даст Radix Dialog перехватить взаимодействия с дропдауном.

### Технические детали

**Файл:** `src/components/ui/EmployeeMultiSelect.tsx`

На div-контейнере дропдауна (`data-employee-dropdown`) добавить обработчики:

- `onPointerDown={e => e.stopPropagation()}` -- предотвращает закрытие Radix Dialog при клике
- `onMouseDown={e => e.stopPropagation()}` -- дублирование для совместимости с обработчиком `handleClickOutside`
- `onTouchStart={e => e.stopPropagation()}` -- для мобильных устройств

Также убрать условие `autoFocus={!usePortal}` и всегда ставить `autoFocus` на поле поиска, чтобы при открытии дропдауна фокус сразу попадал в поиск (в том числе в модалке).

Изменения затрагивают только один файл и не влияют на остальные места использования компонента.
