

## Проблема: новые компании в тендерах пропадают после сохранения

### Диагностика

В базе данных протокола №54 от 16.03 в секции тендеров сохранены только 3 компании (БСК Неглинная, Лаборатория Касперского, ВК единичные расценки), добавленные в 11:36 UTC. Никаких более поздних записей нет.

**Корневая причина**: компании в тендерной секции хранятся не как отдельные сущности, а как префиксы `[Название компании]` в тексте пунктов (`item_text`). Если пользователь добавил новую компанию, но:
1. Не добавил в неё ни одного пункта — компания исчезает при сохранении (нет записей для неё в БД)
2. Добавил пункты с пустым текстом — они фильтруются строкой `items.filter((item) => item.item_text.trim())` ДО добавления префикса компании

### Решение

**Файл: `src/pages/ProtocolEditor.tsx`** — 2 изменения:

1. **Пустые компании**: при сохранении для каждой компании без пунктов автоматически создавать «заглушку» — пункт с текстом только `[CompanyName]` (пустой item_text после парсинга). При загрузке парсер `parseTenderItemsToGroups` уже обработает такой пункт корректно.

2. **Фильтр пустых**: для тендерных пунктов НЕ фильтровать по `item_text.trim()`, потому что после добавления префикса `[CompanyName]` текст станет непустым. Перенести фильтрацию после формирования финального текста, либо учитывать компанию:

```typescript
// В processItemsInParallel для тендеров — не фильтровать пустые,
// потому что [CompanyName] + пустой текст = валидный пункт
const itemsToProcess = isTender 
  ? items  // все пункты тендера сохраняем
  : items.filter((item) => item.item_text.trim());
```

3. **Пустые компании → placeholder-пункт**: при сохранении, если компания имеет 0 пунктов, создать один пункт-маркер:

```typescript
if (group.sectionType === "tender" && group.companyGroups) {
  for (const company of group.companyGroups) {
    const itemsToSave = company.items.length > 0 
      ? company.items 
      : [{ id: generateTempId(), item_text: "", responsible: null, due_date: null, project_id: null }];
    // ... processItemsInParallel с itemsToSave
  }
}
```

### Масштаб
Один файл, 2 точечных правки в логике сохранения/фильтрации тендерных пунктов.
