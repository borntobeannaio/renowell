

## Тендер-протоколы: отдельный тип протокола

### Суть

Создать новый тип протокола — **тендер-протокол**. Он использует существующую таблицу `protocols` (поле `meeting_type` для различения типов). Внутри — только секции типа `tender` (компании как фритекст). В разделе "Протоколы" — вкладки для переключения между обычными и тендер-протоколами.

### Что меняется

**1. Таблица `protocols`** — без изменений схемы. Используем поле `meeting_type`:
- `meeting_type = 'tender'` — тендер-протокол
- Всё остальное — обычный протокол

**2. `src/components/modules/ProtocolsModule.tsx`** — Табы для разделения:
- Добавить `Tabs` (Radix) с двумя вкладками: "Протоколы совещаний" и "Тендер-протоколы"
- Фильтрация `protocols` по `meeting_type`
- Кнопка "Новый тендер-протокол" → `/protocols/new?type=tender`
- Для тендер-протоколов карточки те же, но с иконкой Building

**3. `src/pages/ProtocolEditor.tsx`** — Режим тендер-протокола:
- Считывать `?type=tender` из URL
- Если `type=tender` или редактируемый протокол имеет `meeting_type=tender`:
  - Заголовок по умолчанию: "Тендеры" (можно изменить)
  - Организатор по умолчанию: искать сотрудника "Опарин Андрей" в списке employees
  - `meeting_type` при сохранении: `'tender'`
  - Скрыть кнопку "Добавить секцию" — вместо неё кнопка "Добавить компанию"
  - Все секции имеют `sectionType: 'tender'`, каждая секция = одна компания
  - При создании задач — текст формата `[Название компании] текст задачи`

**4. `src/hooks/useProtocols.ts`** — без изменений, `meeting_type` уже есть в интерфейсе

**5. `src/pages/ProtocolEditor.tsx` — Специфика тендер-режима:**
- Начальное состояние: пустой список секций (каждая секция = компания)
- Кнопка "+ Компания" вместо "+ Секция" → добавляет секцию типа `tender` с фритекст-именем компании
- Метаданные: поле `title` предзаполнено "Тендеры", `meeting_type = 'tender'`
- `SectionTypeModal` не нужен — вместо него простой input для имени компании (модалка с текстовым полем "Название компании")

### Детали по файлам

| Файл | Изменения |
|---|---|
| `ProtocolsModule.tsx` | Добавить Tabs (обычные / тендеры), фильтрация по `meeting_type`, кнопка "Новый тендер-протокол" |
| `ProtocolEditor.tsx` | Чтение `?type=tender`, дефолтные значения для тендер-протокола, скрытие `SectionTypeModal`, кнопка "+ Компания", `meeting_type='tender'` при сохранении |
| Новый: `AddCompanyModal.tsx` | Простая модалка с полем "Название компании" |

### Логика задач

При сохранении пунктов тендер-протокола текст задачи формируется как `[CompanyName] item_text` — это уже реализовано в текущем `handleCreate`/`handleSaveChanges` для секций типа `tender`. Задачи попадают в проект "Тендеры/задачи" (`bf2ef5b4-...`) — тоже уже настроено.

### Дефолтный организатор

При `type=tender` — после загрузки employees найти сотрудника с `full_name` содержащим "Опарин" и "Андрей", подставить его ID в `form.organizer_id`.
