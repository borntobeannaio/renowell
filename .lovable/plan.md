

## Исправление системы снепшотов черновиков

### Проблема
Снепшоты черновиков **не создаются**, потому что триггеры базы данных не были привязаны к таблице `form_drafts`. Функции `save_draft_snapshot()` и `cleanup_old_snapshots()` существуют, но не вызываются. Таблица `form_draft_snapshots` пуста (0 записей).

### Решение

#### 1. Создать триггеры в базе данных

Создать два триггера:
- **BEFORE UPDATE** на `form_drafts` -- вызывает `save_draft_snapshot()`, сохраняет предыдущие данные перед обновлением
- **AFTER INSERT** на `form_draft_snapshots` -- вызывает `cleanup_old_snapshots()`, чистит старые записи (старше 7 дней, более 50 штук)

#### 2. Включить автосохранение для новых протоколов сразу

Файл: `src/pages/ProtocolEditor.tsx`

Сейчас `saveEnabled` требует `hasUnsavedChanges === true`. Для новых и копируемых протоколов автосохранение должно стартовать сразу при открытии редактора, чтобы снепшоты начинали создаваться с первой минуты работы.

Изменение: `saveEnabled: !!user && !isCopyDataLoading && !isEditDataLoading && (editInitialized || isNew) && (hasUnsavedChanges || isNew)`

#### 3. Исправить ошибку сборки

Файл: `src/hooks/usePushNotifications.ts`

Добавить приведение типа для `pushManager` (TS не знает о Web Push API в текущих типах).

### Интервал автосохранения

Остается **3 секунды** (без изменений, как вы просили). Снепшот создается автоматически при каждом обновлении черновика через триггер БД.

### Итог

После этих изменений:
- Каждые 3 секунды черновик обновляется в `form_drafts`
- Триггер автоматически сохраняет предыдущую версию в `form_draft_snapshots`
- Панель "История" в редакторе начнет показывать версии для восстановления
- Старые версии автоматически удаляются (лимит 50 штук / 7 дней)
